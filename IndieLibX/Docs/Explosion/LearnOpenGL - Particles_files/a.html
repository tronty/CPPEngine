            JoeyDeVries Sensei Phil • a year ago

            1) It seems like the vertices for the quad are defined in NDC so when we multiply by scale (10.0f) shouldn't it be clipped for being outside the range [-1, 1]?

            We are indeed multiplying it by a large scale, but before sending it to gl_Position we're multiplying the resulting position with the projection matrix. The orthographic projection we've set up transforms the vector from the game resolution (800x600 I believe) to [-1,1]. So if the resulting vec4((vertex.xy * scale ) + offset, 0.0, 1.0); ends up being something like vec4(500, 300, 0.0, 1.0); it'll end up being vec4(0.25, 0.0, 0.0, 1.0); after the perspective transform, which is written to gl_Position.

            2) It seems like we are mixing coordinate systems for the vertices of quadParticle and offset? Can't see how they are related

            We use quadParticle's NDC coordinates to neatly scale the particle based on some scale value (which transforms the position from local to world) and add an offset to the scaled world-space version.

            3) Why was a model matrix not used to translate the particles?

            We only need a vec2 offset, a mat4 isn't really necessary.
