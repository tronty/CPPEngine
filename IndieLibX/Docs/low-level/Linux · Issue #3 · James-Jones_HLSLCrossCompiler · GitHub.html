James-Jones commented on 23 Sep 2013

I will take a look at Linux support. No reason it should not work although you need Windows to run fxc.exe from Windows SDK in order to compile the HLSL shaders to bytecode. I suppose Wine could do the trick if you wish.
Regarding the .o files, they are intentional. The .asm files are text files showing a human-readable representation of the HLSL, which are extremely helpful when debugging the cross compiler. The .o files are the bytecode files which the cross compiler uses as input to create the GLSL equivalent.
For example - given a HLSL shader SM5 vertex shader in a file called MyShader, run
fxc /T vs_5_0 /Fo MyShader.o /Fc MyShader.asm MyShader
