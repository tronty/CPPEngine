    Managing the Android SDK and NDK on OSX using Homebrew

    Skip to end of metadata
    • Created by Paul Mietz Egli, last modified on Mar 27, 2012
    Go to start of metadata

    The Homebrew package manager is a convenient, clean way to install and update software that isn't part of the basic OSX environment. Homebrew can be used to manage the Android SDK and NDK for use with Titanium Studio and the command line tools:

    1. Install Homebrew according to the instructions on the web site (http://mxcl.github.com/homebrew/).
    2. Open Terminal and run brew install android-sdk, then brew install android-ndk.
    3. Edit ~/.bash_profile and add the following exports:
      ?
      ##
      ## Android SDK
      ##
      export ANDROID_SDK=/usr/local/Cellar/android-sdk/r17
      export ANDROID_NDK=/usr/local/Cellar/android-ndk/r7b
      You may need to change the paths depending on the current version of the SDK or NDK.
    4. In the terminal, run source ~/.bash_profile to get the new variables into your environment.
    5. Run android to start the Android SDK Manager.
    6. Follow the instructions on the Appcelerator wiki for installing the required SDK components.
    7. Launch Titanium Studio and open the Preferences. Select Titanium Studio > Titanium, and set the Android SDK Directory to /usr/local/Cellar/android-sdk/r17/. The trailing slash is significant; I found that the SDKs won't be found without it.

    As new Android SDK and NDK releases come out, simply run brew upgrade to get the latest version. You will need to edit your .bash_profile and Titanium Studio settings for each release as the Homebrew formula for these packages installs them to a path that contains the SDK version number. For example, the previous release of the SDK was in /usr/local/Cellar/android-sdk/r16.
