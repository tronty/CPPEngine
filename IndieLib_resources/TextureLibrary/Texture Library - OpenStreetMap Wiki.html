Technical restrictions

General information

  • Allowed texture size ( width and/or lenght ): 64,128,256 pixel. Exceptionally 512 or 1024 pixels.
  • data format: *.jpg, *.png
  • Put here only seamless (without visible edges when texture repeated) pictures!
  • Some tips how to make pictures for generating of Seamless Textures for this library. 
Note: Phototextures are unique and used for covering of the entire façade 
of important objects without repeating! ( e.g. Notre Dame Paris)

Naming

Syntax idea 1

DE12AB_YourTextureName

  • DE: Shortcut (Like RU, Pl, JP..) of the country from which the image of the texture comes
  • 12: Age - century from the photographed object. If unknown, put 99. If BC put minus: e.g.:-6 means VI BC
  • AB: The number of the type of surface. See: surface table
  • YourTextureName: Your own name of the surface. Preferred five-digit number like "12345" because short and machine friendly, but also every other names are possible. Use, if desired, names in Your country language but without special characters like ą, ö, ø.

Syntax idea 2

There is also the idea to use the following naming scheme for texture files.

CODE_NAMESPACE_NAME_COUNTRY_CENTURY_DIMENSIONS

where: CODE — as now, code from http://wiki.openstreetmap.org/wiki/Surface_table NAMESPACE — author nickname, website, etc. NAME — number in namespace collection

This three components form unique name which is enough for the identifying of the texture.

COUNTRY CENTURY DIMENSIONS — 5mx2.5m. Units are determined by contributor (100ftx50ft is also okay). This components are descriptive part of the name.


Syntax examples, in filename order:

A1_JanV_00001_RU_21_5m_5m.jpg

A8_JanV_00001_RU_20_1.5m_1.5m.jpg

A8_JanV_00002_RU_20_1m_2m.jpg

A8_nanodesu.ru_00001_RU_21_4m_4m.jpg

A8_nanodesu.ru_00002_RU_20_1m_1m.jpg
